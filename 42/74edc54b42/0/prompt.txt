# Handle duplicate code Z-Wave notification (Issue #848)

## Context

When a lock rejects a code because it duplicates a code in another slot, Z-Wave sends an Access Control notification (type 6, event 15: `NEW_USER_CODE_NOT_ADDED_DUE_TO_DUPLICATE_CODE`). LCM currently ignores this notification, so the in-sync sensor sees the code isn't set, retries, gets rejected again, and loops indefinitely.

**Goal**: When this notification is received, disable the affected slot, log an error, and create a persistent notification so the user knows to set a unique code.

## Changes

### 1. `providers/zwave_js.py` — Handle duplicate code notification

**Add `_set_in_progress_code_slot: int | None` field** to `ZWaveJSLock` (default `None`). Lifecycle:
- **Set** in `async_set_usercode` before the service call
- **Cleared on success** in `on_value_updated` (inside `subscribe_push_updates`) when a real value update arrives for the in-progress slot — this confirms the lock accepted the code
- **Cleared on failure** in `_handle_duplicate_code` when the notification is handled

**In `_handle_zwave_js_event`**, before the normal event flow, check for event 15 — but **only when `_set_in_progress_code_slot` is set**. This guards against reacting to duplicate code notifications triggered externally (e.g., someone manually adding a duplicate code at the lock's keypad). We only want to react when LCM initiated the set operation:

```python
if (
    evt.data[ATTR_EVENT] == AccessControlNotificationEvent.NEW_USER_CODE_NOT_ADDED_DUE_TO_DUPLICATE_CODE
    and self._set_in_progress_code_slot is not None
    # If userId is in params, verify it matches our in-progress slot
    # (otherwise the notification is about a different slot, not ours)
    and (code_slot == 0 or code_slot == self._set_in_progress_code_slot)
):
    self._handle_duplicate_code()
    return
```

**In `on_value_updated`** (inside `subscribe_push_updates`), clear the field when a real push update arrives for the in-progress slot (confirms lock accepted the code):

```python
if code_slot == self._set_in_progress_code_slot:
    self._set_in_progress_code_slot = None
```

(Add this early in the handler, before the dedup/skip checks)

**Add `_handle_duplicate_code()` callback method** to `ZWaveJSLock` (no params — uses `_set_in_progress_code_slot` directly since we only enter this when it's set):
- Read `code_slot = self._set_in_progress_code_slot`
- Clear `_set_in_progress_code_slot`
- Find all LCM config entries managing this lock + slot (same pattern as `async_fire_code_slot_event`)
- For each matching config entry:
  - Disable the slot: `data[CONF_SLOTS][slot_num][CONF_ENABLED] = False` via `hass.config_entries.async_update_entry()`
  - Create persistent notification with ID `{DOMAIN}_{entry_id}_{slot}_duplicate_code`
- Log an error once (not per config entry)

**New imports**: `copy`, `from homeassistant.components.persistent_notification import async_create`, `from homeassistant.const import CONF_ENABLED`

### 2. `tests/providers/test_zwave_js.py` — Add tests

Tests following existing notification event test patterns:

1. **`test_duplicate_code_notification_disables_slot`** — Event 15 with `_set_in_progress_code_slot` set, matching userId → slot disabled in config entry, persistent notification created, error logged, field cleared
2. **`test_duplicate_code_notification_no_user_id_disables_slot`** — Event 15 with `_set_in_progress_code_slot` set, no userId in params → still handles it (uses in-progress slot)
3. **`test_duplicate_code_notification_ignored_when_not_in_progress`** — Event 15 arrives but `_set_in_progress_code_slot` is None (external trigger) → notification ignored, no slot disabled
4. **`test_duplicate_code_notification_ignored_when_user_id_mismatches`** — Event 15 with `_set_in_progress_code_slot` set to slot 2, but userId says slot 3 → ignored (not our operation)
5. **`test_set_in_progress_cleared_on_value_update`** — After `async_set_usercode`, a real push value update for that slot clears `_set_in_progress_code_slot`

       ```python
       async def _async_handle_duplicate_code(self) -> None:
           """Handle duplicate code rejection from the lock."""
           code_slot = self._set_in_progress_code_slot
           self._set_in_progress_code_slot = None

           switch_entity_ids: list[str] = []
           affected_entries: list[str] = []
           slot_key = str(code_slot)
           for entry in self.hass.config_entries.async_entries(DOMAIN):
               if self.lock.entity_id not in get_entry_data(entry, CONF_LOCKS, []):
                   continue
               if slot_key not in get_entry_data(entry, CONF_SLOTS, {}):
                   continue
               enabled_uid = f"{entry.entry_id}|{code_slot}|{CONF_ENABLED}"
               entity_id = self.ent_reg.async_get_entity_id(
                   SWITCH_DOMAIN, DOMAIN, enabled_uid
               )
               if entity_id:
                   switch_entity_ids.append(entity_id)
                   affected_entries.append(entry.title or entry.entry_id)

           if switch_entity_ids:
               await asyncio.gather(
                   *(
                       self.hass.services.async_call(
                           SWITCH_DOMAIN,
                           SERVICE_TURN_OFF,
                           {ATTR_ENTITY_ID: entity_id},
                           blocking=True,
                       )
                       for entity_id in switch_entity_ids
                   )
               )

           _LOGGER.error(
               "Lock %s rejected code for slot %s because it duplicates a code "
               "in another slot. The slot has been disabled. Please set a unique "
               "code and re-enable the slot",
               self.lock.entity_id,
               code_slot,
           )

           async_create(
               self.hass,
               (
                   f"Lock **{self.lock.entity_id}** rejected the code for "
                   f"slot **{code_slot}** because it duplicates a code in "
                   f"another slot. The slot has been disabled. "
                   f"Please set a unique code and re-enable the slot."
               ),
               title="Duplicate Lock Code Rejected",
               notification_id=(
                   f"{DOMAIN}_{self.lock.entity_id}_{code_slot}_duplicate_code"
               ),
           )
       ```
